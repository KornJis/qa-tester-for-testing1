## Severity & Priority guide / เกณฑ์ความรุนแรงและลำดับความสำคัญ

**Severity (ผลกระทบ):**
- **S1 — Critical:** blocks core flow / data loss / crash, no workaround.
- **S2 — Major:** important feature broken, awkward workaround exists.
- **S3 — Minor:** small functional or UI issue, easy workaround.
- **S4 — Trivial:** cosmetic, typo, polish.

**Priority (ความเร่งด่วน):** P1 (fix now) / P2 (fix soon) / P3 (fix later).
